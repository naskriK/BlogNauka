08.12.2023 - Dzień 1 - Jak działa git i platforma GitHub

Naucz się ze mną funkcji gita które pozwolą Ci pracować efektywnie w zespole i poza nim.

Czym jest git?

Git jest narzędziem które pozwala nam zarządzać swoim projektem i jego zmianami.

Możemy śledzić co zostało zmienione, dodawać zmienione momenty do pamięci, odwracać tą pamięć. Dlatego bardzo łatwo jest odwrócić coś co zostało zmienione na tyle, że nasza aplikacja przestała działać.

GitHub - jest platformą gdzie możemy zapisywać nasze repozytoria gitowe a także trzymać tam nasze projekty.

Szybki overview na temat Terminala

Terminal będzie nam potrzebny do korzystania z funkcji gita, dlatego rzucę na niego teraz okiem i opiszę działanie a także podstawowe funkcje.

Co to jest Terminal?

Terminal pozwala na pracę z systemem operacyjnym, każdy system ma swój własny terminal, dzięki niemu możesz działać wprost na system. Tworzyć foldery, poruszać się po nich, a także robić bardziej zaawansowane rzeczy.

Terminale w systemach mogą się różnić, nie każda opisana funkcja tutaj będzie działać akurat na twoim terminalu!

Podstawowe funkcje :

  1. Poruszanie się (komenda :) cd

    Pozwala poruszać się po folderach, cd .. schodzi poziom niżej w drzewie a cd (nazwa folderu) pozwala do niego wchodzić. Możemy też użyć cd (ścieżka) to przeniesie nas do konkretnej ścieżki.

  2. Tworzenie folderu (komenda :) mkdir

    Pozwala stworzyć nam folder o danej nazwie.

  3. Usuwanie folderu (komenda :) rd

    Pozwala usunąć folder o danej nazwie.

  4. Tworzenie pliku (komenda :) copy con

    Pozwala stworzyć plik o danej nazwie. Następnie dodajemy zawartość, jak już jest wystarczająco klikamy CTRL+Z i enter.

  5. Podejrzenie pliku (komenda :) type

    Pozwala oglądać plik o danej nazwie.

  6. Usuwanie pliku (komenda :) del

    Pozwala usunąć plik o danej nazwie.

Wszystko na temat git

Git działa na podstawie tematu "working directory". W momencie inicjalizacji repozytorium gita, tworzy się working directory, miejsce w którym każda nasza zmiana w folderze będzie obsługiwana i sprawdzana. Następnie możemy tą zmianę zapisać za pomocą git add do staging area w którym następnie możemy użyć git commit następnie nasze zmiany wędrują, do miejsca w którym możemy nasze zmiany zapisać już do lokalnego repozytorium czyli do miejsca w którym nasze zmiany zostają zatwierdzone i do których możemy potem wracać w razie problemów. Nazywamy to local repository, w momencie w którym chcemy trzymać nasze repozytorium na np. GitHub wtedy wysyłając nasze repozytorium, GitHub obsługuje to jako remote repository. Wtedy możemy działać na swoim repozytorium a repozytorium na GitHubie jest jako miejsce do którego tylko zapisujemy to co już chcemy zapisać w naszym projekcie.

Aby stworzyć repozytorium lokalne w naszym projekcie korzystamy z komendy git init.

Po użyciu tej komendy wszystkie nasze pliki stają się śledzone przez Git czyli każda zmiana jest już zauważalna, w tym momencie możemy już zarządzać naszymi zmianami i repozytorium.

Aby sprawdzić nasze pliki śledzone i każdy etap warto skorzystać z komendy git status.

Możemy dodać nasze pliki do staging area, do miejsca gdzie będą czekały jako pliki które mogą być commitem do repositorum, ale w momencie w którym zmienimy choćby jakiś plik będzie miał status zmodyfikowanego i będzie trzeba go jeszcze raz dodać albo przywrócić. Komenda : git add (plik) git add --all (wszystkie pliki) git restore (przywrócenie pliku)

Aby zapisać nasze wszystkie zmiany jako commit do repozytorium, korzystamy z komendy git commit. Wymaga to od nas podania wiadomości co zostało zmienione albo co zostało dodane. Krótki opis

Możemy sprawdzić kto stworzył commit, kiedy i jaka jest nazwa używając komendy git log albo git log --oneline

Możemy zmieniać gałęzie naszego repozytorium za pomocą komendy git checkout

Możemy wrócić do jakiegoś momentu commitów w naszym projekcie za pomocą komendy git reset i git revert

Możemy wykluczyć jakiś plik aby git go nie śledził tworząc plik .gitignore i dodając jego nazwe do środka

Git Branches

Możemy działać na różnych gałęziach a następnie łączyć je.

Aby stworzyć osobny branch używamy komendy git branch (nazwa)
